# Contribuindo com este Projeto

Antes de tudo, obrigado pelo interesse em contribuir com o meu portfólio! 🙌
Toda ajuda é bem-vinda: **pull requests, abertura de issues, sugestões de novas funcionalidades, correções, feedbacks ou dúvidas**.

## Como Contribuir

1. **Faça um fork do repositório** e crie uma branch para sua modificação:

   ```bash
   git checkout -b minha-nova-feature
   ```
2. **Implemente suas mudanças** (correções de bugs, melhorias de código, ajustes de layout, etc.).
3. **Teste localmente** para garantir que o projeto continua funcionando.
4. **Abra um Pull Request (PR)** explicando claramente o que foi alterado e por quê.

---

## Abertura de Issues

Se você encontrou um **bug**, tem uma **dúvida** ou deseja sugerir uma **nova funcionalidade**, abra uma [issue](../../issues).

* Descreva o problema ou sugestão de forma clara e objetiva.
* Se possível, adicione **prints de tela, logs ou passos para reproduzir**.
* Para vulnerabilidades de segurança, **não abra uma issue pública**. Veja a [Política de Segurança](./SECURITY.md).

---

## Padrões de Código e Boas Práticas

* Utilize **código limpo e organizado**.
* Prefira nomes de variáveis, funções e componentes **descritivos**.
* Sempre que possível, siga as boas práticas do **React**, **Next.js** e **TailwindCSS**.
* Mantenha as dependências atualizadas.
* Evite incluir informações sensíveis no código (use variáveis de ambiente).

---

## Pull Requests

* Certifique-se de que seu PR está atualizado com a branch `main`.
* Descreva detalhadamente as alterações realizadas.
* PRs pequenos e objetivos são mais fáceis de revisar.
* Sugestões de melhorias de acessibilidade, performance e segurança são sempre bem-vindas.
